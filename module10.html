Module 10: Traits

Introduction

Traits are common properties that classes are meant to implement.

Syntax

In Kotlin, traits are implemented using the interface keyword, similarly to how Java also implements traits through the interface keyword.

Traits in Kotlin can have predefined definitions of functions that can be implemented by other classes or can be overridden by classes that implement a trait.

Classes in Kotlin can also implement multiple traits, which is unlike the case with inheritance in Kotlin where a subclass can extend only one superclass.

Traits in Kotlin can also have properties to them that can be implemented by classes.

Traits in Kotlin can also be subtraits of other supertraits, similarly to how a superclass can have subclasses.

Combining composition with traits can also allow us to create even more modular classes that can use components that share a common trait but implement that trait differently.

By having the name property be any object that implements the trait Name, we are able to store a greate variety of data types and build classes that have more variations in their properties.

Advantages and disadvantages

One of the advantages of traits is that one can break up the properties of superclasses into individual traits that implement different methods and properties. What this allows a programmer to do is to be able to write more modular code where one can choose which classes will implement which trait without having to worry about needing to create another superclass or having to modify a superclass due to a subclasss that is an exception that may need to have a method or property added or removed.

One of the issues with traits is that they, like inheritance, also have the diamond problem where if multiple traits have a function with the same name but a different implementation, attempting to implement multiple traits with functions that have the same name but different definitions can create conflicts when defining a class that implements those traits.

Another issue with traits is that one can create so many traits that there is increased complexity with implementing multiple classes that have common traits where one may need to implement a large amount of traits and ensure that each trait is implemented when creating a class.
