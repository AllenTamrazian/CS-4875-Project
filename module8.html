Whenever one is implementing methods or properties in an abstract class in a subclass, one needs to use the override keyword on those properties and methods to implement that they are to be implemented.

Advantages and disadvantages

Unlike C++, Kotlin does not support multiple inheritance and thus a class can only inherit from one class.

One of the advantages of inheritance is that one can easily moden an is-a relationship where different classes can be part of a category which can then share common properties that can be worked with.

However, one of the issues with using inheritance in Kotlin is that classes are final by default in Kotlin, meaning that these classes cannot be extended or used as superclasses. If a class is to be used as a superclass, it would need to use the abstract keyword for abstract classes or if it is a concrete class, it would need to use the open keyword. The individual properties of the classes must also use the open keyword in order to be overrideable.

Moreover, another issue with inheritance in Kotlin and in other programming languages is that changing a superclass can involve the changing of many other subclasses; for example, if one has an entity class for a game and there are many subclasses that inherit from the entity class and one changes the entity class, one would also need to make changes to all the other classes that also inherit from the entity class.

Another pressing issue with abstract classes is the diamond problem, an issue that happens in classes that support multiple inheritance such as C++ where a class that inherits from multiple superclasses can have the two or more superclasses contain properties and methods with conflicting definitions.
