Module 7 - Classes

Introduction

Classes are a blueprint for creating objects. They contain properties (data) and functions (behavior). For example, let's say we want to make a class named Dog. A dog can have properties such as name, breed, and age, but a dog also has behavior such as barking and wagging its tail. Let's jump in and learn more about classes.

Syntax

The syntax for a class definition consists of the word class as shown below. Class is a reserved keyword for Kotlin to identify classes. Note: Keywords cannot be used as a name for a property. A class also has a name, which is Classname as shown below. There are recommended ways to name a class, which will be explained later. There is also the block of code that contains properties and functions of the class. Note: // in Kotlin make that line into a comment. A comment is for the person looking at the code, but to the computer, it is invisible. It is good to leave comments for yourself and others to understand what the code does.

There are several rules for defining classes in Kotlin, some of which are required and some that are suggestions that programmers in Kotlin follow. These rules are:

  • The name of a class should start with a capital letter, be a noun, and have the first letter of each word to be capital. For example, we could have a class named Dog, Cat, or BankAccount.
  • Properties and functions of a class can be made public, where they are visible to other programs. Properties and functions of a class can also be made private to be invisible to other programs. By default, variables are public.
  • A class can have multiple functions with the same name, but with different input properties (parameters). For example, a restaurant waiter sometimes asks what food you want to order, but sometimes the waiter asks more questions such as if you want no onions or extra cheese.

Classes can be quite useful. Classes can be created multiple times to have different instances (versions) and have different parameters for each one. For example, with a Dog class, we can have a Dog that is named Max, is 1 year old, and is a Golden Retriever. We can also have a Dog that is named Daisy, is 4 years old, and is a Border Collie. They are both part of the Dog class, but they have different properties. We can keep track of each instance with a variable name.
