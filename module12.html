Module 12 - Recursion

Introduction

There are times where we want to do something multiple times. We could use a while or for loops, but we can also use something called recursion. Recursion is when a function calls itself continuously. You can think of recursion as a game of telephone. In telephone, everyone does the same action, telling the next person a word from the previous person, but each time, the result is a little bit different than previous. Eventually, the line of people ends and we get an end result. Recursion uses the idea that there are cases that we want to finish and other cases where we want to continue from the previous action.

Syntax

A recursive function has multiple parts to it. The first part is a normal function with input parameters and a return type. These input parameters usually keep track of what value we are on (imagine x in for(x in list)) and/or what is the current amount/list of something (imagine a count or string). The return type is not any different compared to normal functions.

We then have a line that checks if we have reached a base case, which is when we want to stop and return. This is done with an if loop and statement. We can then specify what action we want to do. If we don't want to stop and return, we can do an action in the else statement. If there are multiple points where we want to stop, we can use multiple else if statements as we have covered before.

Examples

Here is a simple example of recursion. A factorial is what we get when we multiply every whole number less than the number we are starting from until 1. So factorial 4, which is written like 4!, would be 4 * 3 * 2 * 1=24. Note: Factorials can't be negative.

In a factorial, we stop when the number is 1 (n==1). We also stop when we start at a negative number (n<1). In both cases we just return n. We combine the statements to make n<=1. We use that as the condition for the if statement when we get the base case. Our action is returning n. Otherwise (where the else statement is), we multiply n by the next smallest number. We will do this constantly until we stop when n<=1.

Here are the steps for this program broken down:

  1. If n is <=1, then return n.
  2. If n is n!=1, find recursiveFactorial(n−1) and move on to Step 3.
  3. Multiply recursiveFactorial(n−1) to n and return that.
